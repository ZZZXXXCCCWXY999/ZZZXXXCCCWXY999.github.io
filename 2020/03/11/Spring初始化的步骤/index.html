Spring初始化的步骤

一、Spring 的初始化说到底就是Ioc容器初始化。

二、Ioc容器的初始化的几个重要步骤

2. 1、定位

2.1.1所谓的定位就是定位文件资源，如我们常配置的classPathApplicationContext.properties文件，FileSystem文件。

2.1.2在spring 中是通过Reader进行定位配置文件

// 根据 Xml 配置文件创建 Resource 资源对象，该对象中包含了 BeanDefinition 的信息
ClassPathResource resource = new ClassPathResource(“application-context.xml”);
// 创建 DefaultListableBeanFactory
DefaultListableBeanFactory factory = new DefaultListableBeanFactory();
//创建 XmlBeanDefinitionReader 读取器，用于载入 BeanDefinition。
// 之所以需要 BeanFactory 作为参数，是因为会将读取的信息回调配置给 factory
XmlBeanDefinitionReader reader = new XmlBeanDefinitionReader(factory);
// XmlBeanDefinitionReader 执行载入 BeanDefinition 的方法，最后会完成 Bean 的载入和注册。
// 完成后 Bean 就成功的放置到 IOC 容器当中，以后我们就可以从中取得 Bean 来使用
reader.loadBeanDefinitions(resource);

2.1.3 在Spring中加载xml文件的配置文件的方式主要有如下几种：
1、XmlBeanFactory

2、ClassPathXmlApplicationContext

3、FileSystemXmlApplicationContext

4、XmlWebApplicationContext

2、加载

2.2 对配置文件的加载，将配置好的文件进行加载并根据配置的原理以及规则对配置进行解析，如各种Bean之间的关系描述（里面包含了各种设计模式）

3、注册

把我们加载的配置文件（包含了各种Bean之间的关系）转化为具体的Bean文件，通过反射(reflect)，工厂模式等方法进行重新的生产(new 出来)，形成一个完整的由Bean主导(干事)的容器（框架）

4、注入（注入不是Ioc容器完成的）

注入的三种方式：

4.1.set 注入

4.2.构造器注入

4.3.接口注入

5.注入之后我们就可以更具我们定义的关系进行各种操作了（CRUD）
